# 06 Post Processing

## Purpose
This notebook is used to evaluate the spatiotemporal findings across multiple models. It focuses on compiling data generated during the training and validation loops.

## How it Works
The script loads the output CSV files and NumPy confusion matrices from the `./results/` directories generated by the model notebooks. It calculates precision, true positive rates, and false positive rates. Furthermore, it aggregates the performance on combined multi-classroom networks (the Spatial portion of the hypothesis).

## Inputs
- NumPy confusion matrices (`.npy`) and metric CSVs located in the respective `./results/{model}/{dataset}` paths.

## Outputs
- Aggregated `.csv` reports comparing temporal snapshots (percentages) and combinations (different SLNs merged together).

## Execution
Run this notebook only after the desired models (like GraphSAGE and baselines) have completed their training and metric-saving runs.
